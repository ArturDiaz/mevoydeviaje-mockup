ScriptCSS v1.2

Un motor de utilidades CSS generado dinámicamente desde el DOM, sin proceso de build ni dependencias.

Funciona de forma similar a Tailwind CSS pero con una diferencia clave: no requiere compilación. Agrega el script al final del <body> y todas las clases de utilidad se generan en tiempo real leyendo los elementos del DOM.

Ventaja principal: Cero configuración. Escribe clases como p-4, flex, md:grid y el motor genera el CSS automáticamente.

CaracterísticaScriptCSSTailwind CSS
Proceso de buildNo requiereRequerido
Clases en runtimeSíNo (JIT offline)

Instalación

Solo necesitas incluir el script al final del <body> de tu HTML:

<!DOCTYPE html>
<html lang="es">
<head>
  <meta charset="UTF-8">
  <title>Mi sitio</title>
</head>
<body>

  <div class="flex jc-b ai-c p-4 gap-4">
    <h1 class="text-8 font-bold">Hola mundo</h1>
    <button class="px-4 py-2 border-r-4">Botón</button>
  </div>

  <!-- Script al final del body, siempre -->
  <script src="scriptCss.js"></script>
</body>
</html>
Importante: El script debe estar al final del <body>, no en el <head>. Necesita que el DOM exista para leer las clases.

Cómo funciona

Al cargarse, el script ejecuta este flujo:

  1. Recorre todos los elementos del DOM con querySelectorAll('*')
  2. Extrae todas las clases únicas encontradas
  3. Clasifica cada clase: si es responsive (md:flex) o base (flex)
  4. Genera las reglas CSS correspondientes
  5. Las inyecta en un <style id="clases-generadas"> en el <head>
  6. Activa un MutationObserver para re-generar si el DOM cambia

Fórmula general de clases

/* Clase base */
{tipo}-{valor}               →  p-4, m-2, text-6

/* Clase responsive */
{breakpoint}:{tipo}-{valor}  →  md:flex, sm:hide, lg:p-8

/* Clase con !important */
!{tipo}-{valor}              →  !p-4, !flex

/* Clase sin valor (toggle) */
{tipo}                       →  flex, grid, hide, relative

Shorthand de 4 lados NEW

Permite especificar los cuatro lados en una sola clase usando la sintaxis {tipo}-{top}-{right}-{bottom}-{left}. El orden es el mismo que el shorthand CSS estándar.

ClaseCSS generado
p-{t}-{r}-{b}-{l}padding: t×0.25em r×0.25em b×0.25em l×0.25em
m-{t}-{r}-{b}-{l}margin: t×0.25em r×0.25em b×0.25em l×0.25em

Ejemplos

ClaseCSS generado
p-2-2-10-2padding: 0.5em 0.5em 2.5em 0.5em
p-4-8-4-0padding: 1em 2em 1em 0
m-0-4-0-4margin: 0 1em 0 1em
lg:p-6-12-6-12padding: 1.5em 3em 1.5em 3em (dentro de lg)
Valor 0: Se genera sin unidad (0, no 0em), igual que el resto de utilidades de espaciado. 
<!-- Más padding abajo que en los otros lados -->
<section class="p-2-2-10-2">...</section>

<!-- Centrar con margin, sin afectar top/bottom -->
<div class="m-0-4-0-4">...</div>

<!-- Responsive: padding compacto en móvil, holgado en desktop -->
<div class="p-2-2-4-2 lg:p-4-8-6-8">...</div>

Grid — plantilla de columnas (cl) UPD

La clase cl define grid-template-columns en el contenedor. Además del clásico cl-{n} con columnas iguales, ahora acepta variantes para columnas que se adaptan al contenido o se distribuyen automáticamente.

ClaseCSS generadoCuándo usarla
cl-{n}repeat(n, minmax(0, 1fr))N columnas iguales que comparten el espacio disponible
cl-autoautoUna sola columna cuyo ancho se adapta al contenido
cl-minmin-contentColumna tan estrecha como su contenido mínimo
cl-maxmax-contentColumna tan ancha como su contenido máximo
cl-fit-{dim}repeat(auto-fit, minmax({dim}, 1fr))Tantas columnas como quepan; las vacías colapsan
cl-fill-{dim}repeat(auto-fill, minmax({dim}, 1fr))Tantas columnas como quepan; reserva espacio aunque estén vacías
auto-fit vs auto-fill: Con cl-fit-* las columnas vacías colapsan y las existentes se estiran para llenar el espacio. Con cl-fill-* las columnas vacías mantienen su espacio mínimo — útil cuando el número de ítems puede crecer dinámicamente.

El valor {dim} acepta las mismas unidades que w

cl-fit-200px
cl-fit-16rem
cl-fit-40 → 10rem
cl-fill-240px

</div>

Grid — plantilla de filas (rw) NEW

Espejo exacto de cl pero para el eje vertical. Define grid-template-rows en el contenedor, estableciendo filas de altura igual que se reparten el espacio disponible.

ClaseCSS generadoCuándo usarla
rw-{n}grid-template-rows: repeat(n, minmax(0, 1fr))N filas iguales que se reparten la altura del contenedor

rw-nonegrid-template-rows: noneSin filas explícitas — el grid las genera implícitamente

Clases responsive UPD

Agrega un prefijo de breakpoint antes de cualquier clase con el separador :. El sistema cuenta con 16 breakpoints organizados en tres grupos: base, max-* y min-*.

Breakpoints base

PrefijoMedia query generadaRango de pantalla
xs:@media (max-width: 550px)Pantallas muy pequeñas (≤ 550px)
sm:@media (max-width: 767px)Móviles (≤ 767px)
md:@media (min-width: 768px) and (max-width: 1023px)Tablets — rango acotado (768px–1023px)
md-max:@media (max-width: 1023px)Todo hasta tablet inclusive (≤ 1023px)
lg:@media (min-width: 1024px)Escritorio (≥ 1024px)
xl:@media (min-width: 1400px)Pantallas grandes (≥ 1400px)
xg:@media (min-width: 1600px)Pantallas extra grandes (≥ 1600px)

Variantes max-* — hasta un límite superior NEW

Aplican en pantallas iguales o menores al límite indicado. Útiles para aplicar estilos en cualquier pantalla por debajo de un breakpoint específico, sin importar el inicio del rango.

PrefijoMedia query generadaAfecta
max-xs:@media (max-width: 550px)≤ 550px

max-md:@media (max-width: 1023px)≤ 1023px (equivalente a md-max:)
max-lg:@media (max-width: 1399px)≤ 1399px — todo excepto xl y xg
max-xl:@media (max-width: 1599px)≤ 1599px — todo excepto xg

Variantes min-* — desde un límite inferior NEW

Aplican en pantallas iguales o mayores al límite indicado. Permiten activar estilos desde un punto exacto sin depender de los alias semánticos.

PrefijoMedia query generadaAfecta
min-sm:@media (min-width: 551px)≥ 551px — excluye solo xs

min-xl:@media (min-width: 1400px)≥ 1400px (equivalente a xl:)
min-xg:@media (min-width: 1600px)≥ 1600px (equivalente a xg:)
¿Cuándo usar max-* vs min-*? Usa max-* cuando quieras limitar un estilo a pantallas pequeñas ("solo aplica hasta tablet"). Usa min-* cuando quieras activar un estilo a partir de cierto tamaño ("aplica desde tablet en adelante"). La diferencia con los alias base (sm:, lg:, etc.) es semántica: los alias tienen rangos predefinidos fijos, mientras que max-* y min-* son explícitos sobre el límite exacto.

Valores negativos NUEVO

Agrega - al inicio de cualquier clase cuya propiedad soporte valores negativos. La sintaxis es idéntica a Tailwind v3.

/* Fórmula */
-{tipo}-{valor}              →  -mt-10, -top-4, -z-1
{breakpoint}:-{tipo}-{valor} →  sm:-mt-6, lg:-top-8
!-{tipo}-{valor}             →  !-mb-4  (negativo + !important)

Propiedades que soportan negativos

GrupoClasesEjemploCSS generado
Margin -m -mx -my -mt -mr -mb -ml -mt-10 margin-top: -2.5rem
Posición -top -left -right -bottom -top-4

Posición soporta unidades explícitas: top, left, right y bottom aceptan valores con unidad directa — -left-4px, -top-50%, -bottom-2rem, -right-10vw. Sin unidad, el valor se escala × 0.25 rem (-left-4 → -1rem).
Regla del cero: -mt-0, -top-0 y cualquier negativo con valor 0 siempre genera 0 sin signo — nunca -0.
Padding no soporta negativos: -p-4, -px-2, etc. no son válidos en CSS. El script los ignorará y no generará ninguna regla.

Implementación interna

El motor detecta el prefijo - en parsearClase(), extrae el tipo base y lo valida contra TIPOS_NEGATIVOS. Si el tipo está permitido, generarCSSParaClase() antepone - al valor CSS calculado.

// Tipos que aceptan negativos (definido en scriptCss.js)
const TIPOS_NEGATIVOS = new Set([
    'm', 'mx', 'my', 'mt', 'mr', 'mb', 'ml',
    'top', 'left', 'right', 'bottom',
    'z', 'gap', 'gap-x', 'gap-y'
]);

Clase especial: w-responsive

Contenedor inteligente que centra el contenido y adapta su ancho según el breakpoint. En pantallas grandes aplica un ancho fijo, evitando que el contenido se estire en monitores anchos.

¿Por qué ancho fijo en lg/xl? Con width: 100% en una pantalla de 1920px, el contenido se estira horizontalmente y se vuelve ilegible. Un ancho fijo de 1024px o 1400px centrado con margin: auto mantiene la lectura cómoda sin importar el tamaño del monitor.

md interno768px – 1023px100%0 1.5remFull-width con más margen interno

Variantes con prefijo responsive

Puedes usar w-responsive con cualquier prefijo de breakpoint para aplicar el contenedor solo en ese rango específico. Esto es útil para componentes que deben estar contenidos únicamente en ciertos tamaños de pantalla.

ClaseMedia querywidthpadding
xs:w-responsive@media (max-width: 550px)100%0 1rem
sm:w-responsive@media (max-width: 767px)100%0 1rem
md:w-responsive@media (min-width: 768px) and (max-width: 1023px)100%0 1.5rem
lg:w-responsive@media (min-width: 1024px)1024px0
xl:w-responsive@media (min-width: 1400px)1400px0
Base vs. prefijo: w-responsive aplica el comportamiento completo en todos los breakpoints progresivamente (un solo elemento que se adapta a todo). Las variantes con prefijo (lg:w-responsive) se activan únicamente dentro de ese rango — el resto de breakpoints no recibe ningún estilo adicional.

Sistema de caché interno

CachéAlmacenaBeneficio
CACHE.parsedClasesResultado de parsear cada clase (tipo, valor, breakpoint)No re-parsea clases ya vistas
CACHE.cssGeneradoString CSS final para cada claseNo re-genera reglas existentes

El MutationObserver observa cambios en atributos class y nodos nuevos con debounce de 100ms. El caché no se borra entre ciclos — solo se generan reglas CSS para clases nuevas que aparezcan en el DOM.

// Limpiar el caché manualmente (casos excepcionales):
CACHE.limpiar();
Solo limpia el caché si cambias clases dinámicamente con JavaScript en formas no estándar. El MutationObserver ya maneja el 99% de los casos automáticamente.

Integración en proyectos reales

ScriptCSS es compatible con cualquier stack web que sirva HTML.

HTML estático

Sitio simple

Incluye el script y escribe clases directamente. Ideal para landing pages o prototipos.

ASP.NET / Razor

Vistas del servidor

Agrega el <script> en _Layout.cshtml. Funciona en Views, Partials y componentes.

PHP / Laravel Blade

Templates del servidor

Agrega en el layout base. Las clases en parciales se recogen automáticamente.

Vanilla JS / SPA

Apps sin framework

El MutationObserver detecta clases inyectadas dinámicamente sin configuración extra.

Patrón 3 — JavaScript dinámico (fetch / HTMX)

// El HTML retornado por la API puede usar clases ScriptCSS libremente.
// El MutationObserver las detecta y genera el CSS automáticamente.
async function cargarTarjetas() {
  const res  = await fetch('/api/tarjetas');
  const html = await res.text();
  document.getElementById('grid').innerHTML = html;
  // No necesitas llamar a ninguna función manualmente.
}
